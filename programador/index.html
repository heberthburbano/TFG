Manual Técnico de Aves de Pinto.

Documentación viva e integral de la plataforma ornitológica, diseñada para ingenieros, arquitectos y mantenedores. Cubre desde modelado de bases de datos NoSQL hasta integraciones de Edge AI.

Arquitectura General: ViraPedia (Aves de Pinto) es una aplicación Android nativa desarrollada en Kotlin bajo Jetpack Compose, con Arquitectura MVI/MVVM, inyección de dependencias (Hilt) y asincronía basada en Coroutines y StateFlow.

Capa de Dominio

Contratos mediante SpeciesRepository y envoltorio genérico Result<T> para canalizar errores sin excepciones disruptivas.

Capa de Datos

Mapeo resiliente asíncrono desde Firestore (`documentToSpecies`), evitando fallos en UI (NullPointerException) si un campo falta localmente.

Estructura de Firebase NoSQL

El ecosistema Backendless de Firebase almacena avistamientos, información biológica estática de las especies y ficheros multimedia en Storage.

Colecciones Principales

  • users/{userId}: Datos OAuth, estadísticas, y subcolección discoveries/.
  • posts/{postId}: Feed social (latitude, longitude, visibility, etc). Subcolección interna likes/.
  • species/{speciesId}: ViraPedia. Nombres en 5 idiomas (ES, EN, FR, DE, ZH), diet, conservationStatus y propiedades taxonómicas completas.
Reglas de Seguridad: La colección species es de solo lectura pública. El scraper no sobreescribe ciegamente para mantener la integridad relacional de la DB.

Integración de APIs y Scrapers

Scripts Python extraen y enriquecen la base de datos de manera automatizada. Herramientas críticas:

eBird, iNaturalist y Xeno-Canto

eBird da la "semilla" (taxonomía base región ES-MD). iNaturalist añade las imágenes HD libres de Copyright (S3) y el estado de conservación. Xeno-Canto suma las audioUrl MP3 de cantos capturados en España.

Deep-Translator

Traduce asíncronamente (ES, FR, DE, ZH) empleando wrappers. Precisa inyectar timeouts de contingencia para evitar baneos de IP por masiva recurrencia sintáctica.

Reconocimiento Asíncrono: BirdNET (Edge AI)

Integración de TensorFlow Lite para identificar cantos 100% offline dentro de BirdNetAudioRecognizer.kt.

Flujo de Inferencia

  1. Grabación nativa de 3 segundos en PCM_16BIT a 48kHz (micrófono). Aislando el trabajo pesado en Dispatchers.IO.
  2. Acondicionamiento: Normalización pura `/ 32768.0f` cargando el bloque en un `FloatArray`.
  3. Inferencia en birdnet.tflite (~25MB), resolviendo el argmax de clases resultantes.

Modelos Generativos: Gemini y Vertex AI

Se aplica LLMs avanzados a nivel Backend (Python) y a nivel cliente Frontend (Kotlin).

Enriquecimiento Python

enrich_bird_details.py: Prompt estricto (JSON output, temperature=0.1) a Gemini 2.5 Flash pidiendo dietas, peso y hábitat de forma masiva. Se manejan los reintentos vía Regex sobre los errores 429 RESOURCE_EXHAUSTED.

Asistente Chat (Kotlin)

Inyección dinámica mediante `systemInstruction`. Obliga a devolver texto libre adornado con un JSON puro invisible oculto por Regex final. Además, usa `content { image(bitmap) }` habilitando un OCR natural para ornitología visual.

Multimodal Security Filter

El Feed bloquea spam subido por usuarios implementando configuraciones severas de moderación (HarmBlockThreshold.LOW_AND_ABOVE). Imágenes fuera del contexto aviar resultan en iteración nula de subida al servidor.

Navegadores y Heatmaps

Subsistemas de localización orientados al avistador ciudadano en la comunidad (MapScreen.kt, FeedScreens.kt, etc).

  • Capa GBIF (Global Biodiversity): Teselas transparentes estacionales (UrlTileProvider) repintadas dinámicamente inyectando mes y `taxonKey` para mostrar el flujo migratorio continental en la ViraPedia.
  • Snapshots en Vivo: Monitorio asíncrono de Firestore (`postsCollection.addSnapshotListener`) de posts orgánicos en el transcurso de las últimas 5 horas y parseados inteligentemente descartando lats/longs nulos (0,0).
  • Navegación Intents: Redirección delegada mediante esquema URI hacia Google Maps nativo (google.navigation:q=lat,lng), liberando de peso a la app en sí.

Configuración del Entorno

El proyecto requiere un entorno específico debido a las múltiples llaves de APIs biológicas y al uso del Edge AI (TFLite).

Requisitos del Sistema

  • IDE: Android Studio Jellyfish (o superior).
  • JDK: JDK 17 (Integrado en el proyecto).
  • SDK de Android: Mínimo API 24, Target SDK 34+.

Secretos y Variables (`local.properties`)

El manifiesto (AndroidManifest.xml) requiere un token válido para el SDK de mapas que deberá inyectarse en el local.properties:

MAPS_API_KEY=AIzaSyTuClaveSecretaDeGoogleMaps...

Integración Firebase (`google-services.json`)

Debes colocar el archivo google-services.json en la raíz del módulo app/ para que el generador de Hilt/Firebase provea los repositorios de la base de datos (Firestore) y del sistema Auth.

Guía de Compilación

El compilador asume un acoplamiento puro con `gradlew` y módulos inyectados.

1. Limpieza y Reconstrucción

Tras inyectar los Secrets de las APIs y Firebase, ejecuta ./gradlew clean build para forzar a Hilt a re-generar todos los grafos de dependencias Dagger subyacentes.

2. Ejecución (Debug)

Haz click en Run 'app'. Recomendamos encarecidamente testear la inferencia de Puntos de Navegación y Grabación `PCM_16BIT` en un dispositivo físico, ya que el emulador virtual de Android a menudo no simula las frecuencias del micrófono requeridas por BirdNET.

Cuidado con los Permisos: Asegúrate de que el manifiesto solicita activamente los permisos RECORD_AUDIO, ACCESS_FINE_LOCATION e INTERNET al ejecutar por primera vez.
